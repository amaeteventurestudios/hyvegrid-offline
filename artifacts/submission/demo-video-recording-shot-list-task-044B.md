# Task 044B Demo Video Recording Shot List

## Purpose

Provide a short recording shot list for the HyveGrid Offline Devpost demo video, based on the verified Task 042A demo script and Task 044A submission readiness artifacts.

## Recording target

Record a concise 5 to 7 minute demo showing the local app at localhost, the main Hive Health Advisor flow, retrieved local sources, Completed locally status, language selector, Offline System Status, and safety limitations.

## Shot list

| Order | Screen | Action | Spoken point |
|---:|---|---|---|
| 1 | Browser URL bar and Mission Control | Show `http://127.0.0.1:8000` | HyveGrid Offline runs locally at localhost. |
| 2 | Mission Control | Show advisor cards | Five advisor areas support hive health, site readiness, harvest quality, forage and pollination, and hive signal checks. |
| 3 | Hive Health Advisor | Paste the official Hive Health prompt | The app accepts field observations and prepares local guidance. |
| 4 | Waiting state | Submit and wait | The answer path uses a local GGUF model through llama.cpp and local retrieved apiculture notes. |
| 5 | Returned answer | Show answer body | The answer is cautious field triage, not final confirmation. |
| 6 | Source cards | Show retrieved sources | Retrieved sources come from local public apiculture notes. |
| 7 | Completion status | Show `Completed locally.` | The answer completed locally. |
| 8 | Site Readiness Advisor | Show or mention second official prompt | Site readiness supports hive placement questions before installation. |
| 9 | Language selector | Show English, Yorùbá, Hausa, Swahili | English is default, Yorùbá is supported, and Hausa/Swahili are structured field-template modes needing human review. |
| 10 | Offline System Status | Show local status page | The demo is local and does not require a cloud model or external API during judged runtime. |
| 11 | Closing screen | Return to Mission Control or answer page | Say the safety limitation clearly. |

## Required proof points

- localhost
